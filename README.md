## 1. Генерация SSH-ключа на локальной машине

Ключ создаётся на той машине, **с которой** вы будете подключаться к серверу.

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Просмотр публичного ключа:

| ОС            | Команда                              |
| ------------- | ------------------------------------ |
| Linux / macOS | `cat ~/.ssh/id_ed25519.pub`          |
| Windows       | `Get-Content ~/.ssh/id_ed25519.pub`  |

**Linux / macOS:**

```bash
cat ~/.ssh/id_ed25519.pub
```

**Windows (PowerShell):**

```powershell
Get-Content ~/.ssh/id_ed25519.pub
```

---

## 2. Передача ключа на сервер

Скопируйте публичный ключ на Linux-сервер:

```bash
ssh-copy-id root@<ip_address_сервера>
```

> **Примечание.** Замените `<ip_address_сервера>` на реальный IP-адрес вашего сервера.

---

## 3. Настройка SSH-сервиса на сервере

Откройте конфигурационный файл `sshd`:

```bash
nano /etc/ssh/sshd_config
```

Рекомендуемые параметры для безопасного подключения:

```sh
# Сменить стандартный порт 22 на нестандартный
Port 2983

# Вход по публичному ключу
PubkeyAuthentication yes

# Запретить вход по паролю (только по ключу)
PasswordAuthentication no
PermitEmptyPasswords no

# root может входить ТОЛЬКО по ключу (пароль для root запрещён)
PermitRootLogin prohibit-password

# Исключение: для пользователя user разрешаем вход по паролю
Match User user
    PasswordAuthentication yes

# Отключить интерактивную аутентификацию
KbdInteractiveAuthentication no

# Отключить лишнее (по желанию, для безопасности)
X11Forwarding no
```

> **Важно.** Блок `Match User` всегда должен располагаться в **конце** файла —
> все директивы после него действуют только в рамках совпадения.

---

## 4. Проверка конфигурации

После внесения изменений выполните проверку перед перезапуском сервиса:

```bash
# Проверка синтаксиса конфигурации
sshd -t

# Итоговые настройки пароля для пользователя user
sshd -T -C user=user | grep -i password

# Для root значение должно быть "no"
sshd -T -C user=root | grep -i password
```

Если ошибок нет — примените изменения, перезапустив SSH-сервис:

```bash
systemctl restart ssh
```

> **Внимание.** Не закрывайте текущую сессию, пока не убедитесь,
> что новое подключение по ключу и новому порту работает корректно.
